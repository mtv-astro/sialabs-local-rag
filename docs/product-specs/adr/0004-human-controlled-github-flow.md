# ADR 0004 — Desenvolvimento com controle humano e GitHub rastreável

## Status

Proposto

## Contexto

O projeto é uma peça de portfólio. O processo de construção precisa ser tão demonstrável quanto o código.

## Decisão

Usar fluxo incremental com issue, branch, validação local, commit, push, PR, CI, revisão e merge controlado.

## Consequências positivas

- Histórico profissional.
- Evidência de disciplina técnica.
- PRs pequenos e revisáveis.
- CI como confirmação final.

## Consequências negativas

- Mais etapas do que codar direto na main.
- Exige disciplina de documentação.

## Mitigação

Usar templates, backlog pré-definido e comandos PowerShell documentados.
